## Decisions Made

- **JWKS over static public key:** Use Clerk's JWKS endpoint to automatically rotate keys; simplifies key management.
- **Clerk middleware matcher:** Exclude static files and API routes to avoid interfering with backend API calls.
- **Token extraction pattern:** Use `auth().getToken()` in server components to pass JWT to client components for API calls.
- **Remove NextAuth runtime usage:** Current frontend runtime no longer uses NextAuth routes/config.

## User Setup Required

**External services require manual configuration.** To use Clerk in production:

1. Create a Clerk application at [clerk.com](https://clerk.com)
2. Set `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` in `.env.local` (frontend) and `.env` (backend)
3. Configure OAuth providers (Google) in the Clerk dashboard
4. Update the `CLERK_PUBLISHABLE_KEY` in backend settings to match the publishable key for JWT audience validation

Verification commands:
- Backend: `uv run pytest tests/test_auth.py -k "clerk"` (no tests currently)
- Frontend: `pnpm build` (should succeed with dummy keys)
