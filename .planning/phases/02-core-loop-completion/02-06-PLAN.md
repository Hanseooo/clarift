<task type="auto" tdd="true">
  <name>task 3: Update API client configuration (if needed)</name>
  <files>frontend/src/lib/api-client.ts</files>
  <behavior>
    - Test 1: API client exports a configured client with correct base URL
    - Test 2: Client includes authentication header interceptor
  </behavior>
  <read_first>
    - frontend/src/lib/api-client.ts (existing client)
    - frontend/src/lib/api-types.ts (generated types)
    - docs/dev/architecture.md (API client patterns)
    - Clerk documentation (if needed for auth interceptor)
  </read_first>
  <acceptance_criteria>
    - API client is configured with correct base URL (from environment variable)
    - Client includes authentication header interceptor (Clerk token)
    - TypeScript compilation passes
  </acceptance_criteria>
  <action>
1. Examine `frontend/src/lib/api-client.ts`. It likely imports `createClient` from `openapi-fetch` and uses generated types.
2. Ensure the client configuration includes the correct base URL (should be from environment variable `NEXT_PUBLIC_API_URL` or default).
3. Ensure the client includes an authorization header interceptor that adds the Clerk token (if using Clerk). This should already be present.
4. If the client doesn't exist, create it:
   ```typescript
   import createClient from 'openapi-fetch';
   import type { paths } from './api-types';

   export const client = createClient<paths>({
     baseUrl: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000',
     headers: {
       'Content-Type': 'application/json',
     },
   });
   ```
   Add authentication interceptor using Clerk's `useAuth()` hook (for client-side) or server-side token. This may already be elsewhere.

5. Run `npx tsc --noEmit` again to ensure no type errors.
6. Commit changes.
  </action>

  </verify>
